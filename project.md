# Drug Interaction Knowledge Graph

A system that stores drug interactions in a graph database and answers natural language questions like "Can I take Aspirin with Warfarin?" — using only verified graph data, never hallucinated answers.

---

## What It Does

You type a question in plain English. The system:
1. Reads your question and identifies the drug names in it
2. Looks up those drugs in a Neo4j graph database
3. Returns whether an interaction exists, with a reason — or clearly says none was found

It will never make up an interaction. If it is not in the database, it says so.

---

## Why a Graph Database?

Drug interactions are naturally a network: Drug A interacts with Drug B, which also interacts with Drug C. A graph database (Neo4j) models this directly as nodes and edges, making queries like "find all drugs that interact with Aspirin" fast and intuitive — no joins, no complex SQL.

```
(Ibuprofen) --[:INTERACTS_WITH]--> (Aspirin) --[:INTERACTS_WITH]--> (Warfarin)
```

## Tech Stack

| Layer | Technology | Purpose |
|---|---|---|
| Graph DB | Neo4j | Stores drugs and interactions |
| LLM | Ollama (llama3.2) | Extracts drug names from natural language |
| Backend API | FastAPI | REST endpoints for the frontend and CLI |
| Frontend | Streamlit | Web UI for queries |
| Runtime | Python 3.10+ | Application code |

Everything runs locally. No cloud services required.

---

## Graph Schema

**Nodes:**
```
(:Drug {name: "Aspirin"})
```

**Relationships:**
```
(:Drug)-[:INTERACTS_WITH {reason: "..."}]->(:Drug)
```

The `reason` property on the relationship explains why the interaction is dangerous. This is stored in the graph — not generated by the LLM at query time.

## Seed Data

The database is pre-loaded with 5 drugs and 2 known interactions:

**Drugs:**
- Aspirin
- Warfarin
- Ibuprofen
- Paracetamol
- Metformin

**Interactions:**

| Drug A | Drug B | Reason |
|---|---|---|
| Aspirin | Warfarin | Both affect blood clotting. Combining them significantly increases bleeding risk. |
| Ibuprofen | Aspirin | Both are NSAIDs. Combining increases GI side effects, ulcer risk, and reduces Aspirin's cardioprotective effect. |

## How Each File Works

### `setup_graph.py`
Connects to Neo4j and runs Cypher commands to create drug nodes and interaction relationships. Run this once before anything else.

### `agent.py`
The brain of the system. Three responsibilities:
- **`extract_drugs(query)`** — sends the user's question to Ollama and asks it to identify drug names. Falls back to simple text matching if Ollama is unavailable.
- **`check_interaction(drug1, drug2)`** — queries Neo4j in both directions (A→B and B→A) so order does not matter.
- **`answer_query(query)`** — combines the above and formats the final response.

### `cli.py`
A simple read-eval-print loop. Type a question, get an answer. Type `quit` to exit.

### `api.py`
FastAPI server with three endpoints:

| Method | Endpoint | What it does |
|---|---|---|
| GET | `/health` | Check if the API is running |
| GET | `/drugs` | List all drugs in the database |
| POST | `/query` | Natural language question → answer |
| POST | `/check` | Direct drug1 + drug2 → interaction result |

Interactive API docs available at `http://localhost:8000/docs` once running.

### `frontend.py`
Streamlit web app with two tabs:
- **Natural Language Query** — type or pick an example question
- **Direct Check** — select two drugs from dropdowns

The sidebar shows all drugs currently in the database.

## Setup and Running

### Prerequisites

- Python 3.10+
- Neo4j running locally on `bolt://localhost:7687`
- Ollama installed with llama3.2 pulled: `ollama pull llama3.2`

### Install

```bash
cd ~/Knowledge\ Graph
cp .env.example .env
# Edit .env if your Neo4j password is not "password"
pip install -r requirements.txt
```

### Run (all at once)

```bash
./start.sh
```

This seeds the database, starts the API on port 8000, and starts the Streamlit UI on port 8501.

### Run (step by step)

```bash
# Step 1: Seed the database (run once)
python3 setup_graph.py

# Step 2: Start the API
python3 api.py

# Step 3: Start the UI (in a second terminal)
streamlit run frontend.py

# Or use the CLI instead of the UI
python3 cli.py
```

---

## Safety Rules

- The system only reports interactions that are stored in the graph.
- It never generates or infers interactions using the LLM.
- Every response includes a disclaimer to consult a healthcare professional.
- The LLM is used only for understanding natural language — not for medical knowledge.

## Extending the Project

**Add more drugs:**
Edit `setup_graph.py` and add a new `MERGE` statement:
```cypher
MERGE (d:Drug {name: "Lisinopril"})
```

**Add more interactions:**
```cypher
MERGE (aspirin)-[:INTERACTS_WITH {reason: "..."}]->(lisinopril)
```

Then re-run `python3 setup_graph.py` (MERGE is idempotent — safe to run multiple times).

**Add a new endpoint:**
Add a route in `api.py` and a corresponding UI element in `frontend.py`.
